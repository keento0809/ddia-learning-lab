# i18n 規約

- content/ja/** と content/en/** はファイルパス=slugで1:1対応。片方だけの
  作成・変更・削除は禁止(必ず対で行う)
- UI文言は messages/ja.json と messages/en.json の両方に同一キーで追加する。
  キー規約: 名前空間.要素.状態(ICU MessageFormat)
- 演習YAMLの tests はロジック共有(両言語でハッシュ一致必須)。文言のみ
  {ja, en} 併記
- EN側frontmatterには sourceHash を付与し、JA原文更新時に追随漏れを
  CIが検知できるようにする
- 用語は content/glossary.yaml を正とする。新出用語は必ずglossaryに
  ja/en両方を追加してから本文で使う
- 検証: 変更後は npm run validate:content を実行し、出力を表示する
