# docs/design — 文書一覧と優先順位

| # | 文書 | 役割 |
|---|---|---|
| 01 | 基本設計書 | 仕様の正(要件・カリキュラム・アーキテクチャ) |
| 02 | 詳細設計書 | 仕様の正(画面・API・DB・実行エンジン) |
| 03 | 実装タスク分割書 | タスク定義の正(スコープ・受入基準・Out of Scope) |
| 04 | AI実装指示フロー | 運用の基礎(完了報告形式・レビュー観点) |
| 05 | ADR-007 | インフラ決定(Cloudflare)と設計変更 |
| 06 | 並列実装戦略 | Wave計画・Agent view運用 |
| 07 | 人間作業チェックリスト | 人間の担当作業 |
| 08 | ループ・ハーネス設計書 | 運用の最上位(/goal・評価者・スキル計画) |
| 09 | ADR-008 サーバ分割計画 | 2 Worker構成・サイズ予算・T-501〜505 |

## 優先順位ルール
- 仕様の矛盾: 01/02 > 実装。逸脱は docs/tasks/STATUS.md 決定事項ログに記録された場合のみ有効
- 運用の矛盾: **08 > 06 > 04**(タスク投入は08のT2v2形式、修正ループは/goal方式が正)
- インフラ関連: 05(ADR-007)が01/02の該当箇所を上書き
- サーバ構成・サイズ予算: **09(ADR-008)が05と02の該当箇所を上書き**(2 Worker構成が正)
